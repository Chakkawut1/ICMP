ICMP Overview

Internet Control Message Protocol

ICMP คืออะไร?

โพรโทคอล ICMP คืออะไร? ICMP (Internet Control Message Protocol) เป็นโพรโทคอลที่ใช้ในการสื่อสารและจัดการข้อผิดพลาดในเครือข่ายอินเทอร์เน็ต ซึ่งทำงานอยู่ที่เลเยอร์ที่ 3 ของ OSI Model (Network Layer) โพรโทคอลนี้ไม่ได้ใช้เพื่อรับส่งข้อมูลจากผู้ใช้ แต่เพื่อส่งข้อความแจ้งเตือนเกี่ยวกับสถานะและข้อผิดพลาดในเครือข่าย เพื่อให้แน่ใจว่าอุปกรณ์เครือข่ายต่าง ๆ เช่น เราท์เตอร์และโฮสต์ (เครื่องเซิร์ฟเวอร์หรือเครื่องลูกข่าย) สามารถทำงานร่วมกันได้อย่างถูกต้อง

หน้าที่ของ ICMP

  • แจ้งเตือนข้อผิดพลาด (Error Reporting): เมื่อมีปัญหาในการส่งข้อมูลในเครือข่าย เช่น ปลายทางไม่สามารถเข้าถึงได้หรือแพ็กเกจข้อมูลถูกบล็อก ICMP จะส่งข้อความแจ้งเตือนกลับไปยังต้นทางเพื่อให้ทราบว่าเกิดข้อผิดพลาด
  • การทดสอบการเชื่อมต่อ (Diagnostic Tools): ใช้ในเครื่องมืออย่าง ping และ traceroute เพื่อทดสอบการเชื่อมต่อในเครือข่าย ตัวอย่างเช่น ping จะส่ง ICMP Echo Request ไปยังปลายทางเพื่อทดสอบว่าปลายทางยังใช้งานได้หรือไม่
  • การจัดการเส้นทาง (Router Signaling): ICMP ช่วยในการบอกสถานะของเส้นทางที่ใช้ในเครือข่ายและแจ้งข้อมูลเส้นทางที่ดีกว่าในการส่งแพ็กเกจข้อมูล

ภาพประกอบ

ขั้นตอนการทำงานของคำสั่ง ping ด้วย ICMP

1. เครื่องต้นทางส่ง ICMP Echo Request ไปยังปลายทาง

2. ปลายทางรับแพ็กเกจแล้วส่ง ICMP Echo Reply กลับมา

3. ต้นทางวิเคราะห์เวลาที่ใช้ในการเดินทางและแจ้งผลให้ผู้ใช้ทราบ

ในภาพจะเห็นการทำงานของ ICMP ที่ส่งคำขอ (Echo Request) จากต้นทางไปยังปลายทาง แล้วได้รับคำตอบ (Echo Reply) เพื่อยืนยันว่าการเชื่อมต่อยังทำงานได้ปกติ

การทำงานของ ICMP

1.การส่งข้อความ ICMP : เมื่ออุปกรณ์ในเครือข่ายต้องการแจ้งข้อผิดพลาดหรือสถานะอื่น ๆ จะส่งข้อความ ICMP ซึ่งมีประเภทหลายรูปแบบ เช่น:

- Echo Request / Echo Reply: ใช้ในคำสั่ง ping เพื่อทดสอบการเชื่อมต่อ

- Destination Unreachable: แจ้งว่าปลายทางไม่สามารถเข้าถึงได้

- Time Exceeded: แจ้งว่าแพ็กเกจข้อมูลหมดเวลาในระหว่างการส่ง (TTL หมด)

2.การตอบกลับ : เมื่อโฮสต์ได้รับข้อความ ICMP จะส่งคำตอบกลับไปเพื่อแจ้งผลการตรวจสอบหรือการเชื่อมต่อ

ตัวอย่างการใช้งาน

- การใช้คำสั่ง ping ส่ง ICMP Echo Request ไปยังปลายทาง (เช่น เว็บไซต์) แล้วรอรับ ICMP Echo Reply กลับมาเพื่อยืนยันว่าการเชื่อมต่อทำงานได้ปกติ

- การใช้ traceroute เพื่อส่ง ICMP และวิเคราะห์เส้นทางที่แพ็กเกจเดินทางไปยังปลายทาง

ข้อดีและข้อจำกัด

ข้อดี

- ตรวจสอบการเชื่อมต่อ: ใช้เครื่องมือเช่น ping และ traceroute เพื่อตรวจสอบสถานะการเชื่อมต่อและระยะเวลาตอบสนอง

- แจ้งเตือนปัญหา: รายงานข้อผิดพลาด เช่น ปลายทางไม่สามารถเข้าถึงได้ หรือเวลาส่งข้อมูลเกิน

- ช่วยในการเลือกเส้นทาง: ช่วยเราเตอร์ในการตัดสินใจเส้นทางที่ดีที่สุดสำหรับการส่งข้อมูล

- ใช้งานง่าย: เครื่องมือเช่น ping ใช้คำสั่งง่ายในการตรวจสอบเครือข่าย

ข้อจำกัด

- เสี่ยงต่อการโจมตี: อาจถูกใช้ในการโจมตีเครือข่าย เช่น Ping of Death หรือ Smurf Attack

- อาจถูกบล็อก: บางอุปกรณ์หรือไฟร์วอลล์อาจบล็อกการส่ง ICMP

- ข้อมูลจำกัด: ให้ข้อมูลพื้นฐานเกี่ยวกับข้อผิดพลาด แต่ไม่ให้รายละเอียดเพิ่มเติมเกี่ยวกับสาเหตุหรือวิธีแก้ไข

- ไม่เหมาะสำหรับการรับส่งข้อมูลทั่วไป: ใช้สำหรับการสื่อสารข้อผิดพลาดและการตรวจสอบสถานะเท่านั้น
